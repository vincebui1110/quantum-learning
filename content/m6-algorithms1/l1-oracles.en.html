The last modules built the machinery: qubits, gates, entanglement. Now we ask the question that made quantum computing famous - when can a quantum computer beat a classical one, and by how much? The cleanest answers come from a simple setting called the query model, and they all lean on one recurring trick: phase kickback. This lesson sets up both.

The oracle: a function you can only ask

Imagine a function \(f\) that turns an \(n\)-bit input into a single bit, \(f:\{0,1\}^n \to \{0,1\}\). We are not told a formula for \(f\). Instead we are handed a black box - an oracle - that we can query: feed it an input, get back the output. The only cost we count is the number of queries. Everything else (preparing states, applying Hadamards, measuring) is "free." This is the query complexity way of measuring an algorithm, and it is where the first provable quantum speed-ups live.

A quantum computer cannot use a box that simply overwrites its input with \(f(x)\), because that would not be reversible - and every quantum gate must be a reversible unitary. The standard fix uses a second register, the target, and stores the answer with an XOR:

\[ U_f\,|x\rangle|y\rangle = |x\rangle\,|y \oplus f(x)\rangle. \]

Here \(|x\rangle\) is the \(n\)-qubit input register, \(|y\rangle\) is a single target qubit, and \(\oplus\) is addition mod 2 (XOR). Applying \(U_f\) twice returns \(y\) to its start, confirming the operation is its own inverse - a genuine unitary.

Why the XOR

Set \(y=0\): then \(U_f|x\rangle|0\rangle = |x\rangle|f(x)\rangle\), the "obvious" behaviour. The XOR form is the reversible version that also works when \(y=1\) and, crucially, when \(y\) is in a superposition - which is the whole point.

The trick that powers everything: phase kickback

Instead of feeding the target qubit a definite \(0\) or \(1\), prepare it in the minus state \(|-\rangle = \tfrac{1}{\sqrt2}(|0\rangle - |1\rangle)\). Watch what \(U_f\) does to a single input basis state \(|x\rangle\):

\[ U_f\,|x\rangle|-\rangle = |x\rangle\,\tfrac{1}{\sqrt2}\big(|0 \oplus f(x)\rangle - |1 \oplus f(x)\rangle\big). \]

Now split into cases. If \(f(x)=0\), the target is unchanged: \(\tfrac{1}{\sqrt2}(|0\rangle-|1\rangle) = |-\rangle\). If \(f(x)=1\), the two labels swap: \(\tfrac{1}{\sqrt2}(|1\rangle-|0\rangle) = -|-\rangle\). Both cases fold into a single tidy formula:

\[ U_f\,|x\rangle|-\rangle = (-1)^{f(x)}\,|x\rangle\,|-\rangle. \]

The target register comes out exactly as it went in - still \(|-\rangle\) - but the value \(f(x)\) has jumped onto the input register as a phase \((-1)^{f(x)}\). This is phase kickback: because \(|-\rangle\) is an eigenstate of the bit-flip \(X\) with eigenvalue \(-1\), XOR-ing anything into it just multiplies the whole term by \(\pm 1\).

Worked step

Take \(n=1\) and query the superposed input \(|+\rangle = \tfrac{1}{\sqrt2}(|0\rangle+|1\rangle)\), with the target in \(|-\rangle\). By linearity we apply kickback term by term:

\[ U_f\,|+\rangle|-\rangle = \tfrac{1}{\sqrt2}\Big((-1)^{f(0)}|0\rangle + (-1)^{f(1)}|1\rangle\Big)|-\rangle. \]

A single query has stamped both values \(f(0)\) and \(f(1)\) into the signs of one input state. No classical single evaluation sees two outputs at once - this parallel-in-phase encoding is the raw material every algorithm in this module refines.

Key idea

With the target in \(|-\rangle\), the oracle acts as \(|x\rangle \to (-1)^{f(x)}|x\rangle\). The function's output becomes a relative phase on the input register. Interference will later turn those phases into a measurable answer.

Query complexity vs gate complexity

Be honest about what "speed-up" means here. The query model counts oracle calls. A one-query quantum algorithm may still use many ordinary gates around that single call, and building a real oracle for a real function itself costs gates. So a dramatic query separation is not automatically a dramatic runtime win. Keep the two ideas separate: query complexity (how many times we ask the box) and gate complexity (total elementary operations). The algorithms ahead give clean, provable query separations; whether they translate into practical speed depends on how cheaply the oracle can be built.

Key takeaways

  • An oracle is a reversible black box: \(U_f|x\rangle|y\rangle = |x\rangle|y\oplus f(x)\rangle\).
  • The query model counts oracle calls only; this is where the first provable quantum advantages appear.
  • Phase kickback: with the target in \(|-\rangle\), \(U_f|x\rangle|-\rangle = (-1)^{f(x)}|x\rangle|-\rangle\).
  • Query complexity is not the same as gate complexity - a query separation is a provable statement, not automatically a wall-clock speed-up.
